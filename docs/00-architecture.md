# 00 — Architecture & design system

The single source of truth for the data model, the visual system, and the shared
features every tab reuses. Read before any tab doc.

## 1. Stack

- **One self-contained `index.html`** — no framework, no build step. Vanilla JS,
  one module, scoped helpers. Chart.js 4 via CDN (`cdn.jsdelivr.net`).
- **`data.js`** — generated by `scripts/build_data.py` from `data/con-bill-6mon-*.csv`.
  Never hand-edit; re-run the script after CSV changes.
- Fonts via Google Fonts CDN. Offline fallback: tables + all non-chart UI work;
  charts degrade to system fonts.

## 2. Data model (`data.js`)

```js
const DATA = {
  generated:  "2026-07-08",            // build stamp (passed in; Date not used in-script)
  snapshots:  ["20260707","20260708"], // sorted ascending; last = latest
  latest:     "20260708",
  months:     ["2026-02",...,"2026-07"],// latest 6 distinct, ascending
  schemes:    ["AD",...],              // 25, sorted
  trustees:   ["AIA",...],             // 12, sorted
  statuses:   ["OPEN","PARTIAL_SUBMIT","SUBMITTED","APPROVED",
               "PARTIAL_PAID","FULLY_PAID","CLOSED","OVERPAID",
               "REFUND_OVERPAID","WAIVED"],            // canonical lifecycle order
  modes:      ["REGULAR","LUMP_SUM","SURCHARGE"],
  freqs:      ["MONTHLY",...],         // sorted by frequency, blank → "(unset)"
  acctTypes:  ["TVC","SVC","REE","CEE","PAH","SEP"],
  channels:   ["DDE","BATCH","PORTAL","BULKUPLOAD","OTHER"],
  rows: [
    { s:"20260708", tr:"BCT", sc:"AD", st:"APPROVED", bm:"REGULAR",
      fq:"MONTHLY", at:"TVC", ym:"2026-07",
      bill:9, ontime:8, total:8, dde:0, batch:0, portal:0, bulk:0, other:8 }, ...
  ]
}
```

- Every measure is an **int** (pipeline coerces `9.0` → `9`).
- Only the **latest 6 YEAR_MONTHs** are kept; older rows are dropped in the pipeline.
- `statuses` is kept in **canonical lifecycle order** (not alphabetical) so
  status charts read left-to-right as a lifecycle.

## 3. Aggregation helper (shared)

A single `groupBy(rows, keyFn, accumFn)` reduces the filtered row set by any key.
Every tab calls it with its own key (scheme, trustee, status, …). Measures are
summed; derived metrics are computed **after** summing (ratio of sums, never mean
of ratios):

- on-time rate = `Σontime / Σtotal`
- coverage     = `Σtotal / Σbill`
- channel share = `Σchannel / Σ(channels)`

## 4. Global filter bar (top of content)

```
[ snapshot ▼ ]  [ scheme ▼ ]  [ month range ]  [ mode: Current | Compare | Trend ]
```

- **snapshot** — defaults to `DATA.latest`. Drives Current + Trend modes.
- **scheme** — defaults to *All*. Every tab is scoped to it (this is what makes the
  app "scheme_code based"). Single-scheme selection also narrows every chart.
- **month range** — defaults to all 6; lets a user focus a sub-window.
- **mode** — the cross-tab feature toggle:
  - **Current** — show the selected snapshot's numbers (default).
  - **Compare** — pick snapshot A + B; show Δ = B − A. Diverging blue↔red.
  - **Trend** — line across the 6 months (within the selected snapshot).
  - Tabs where a mode is meaningless hide that option (see each tab doc).

## 5. Design system

### Aesthetic — "editorial data observatory"

Refined, paper-and-ink, serious. Deliberately **not** generic AI: no Inter, no
purple-on-white gradients. Generous whitespace, hairline rules, a single
display serif for chrome only.

### Theme — single light theme

Warm off-white surfaces, near-black ink. (Dark was rejected: the categorical set
hits the CVD floor band on the dark surface; light keeps the palette cleanest and
the per-tab data table is the contrast-relief channel by construction.)

### CSS variables (roles, not raw hex, in chart + UI)

```css
:root{
  /* surfaces & ink */
  --page:#f4f2ec; --surface:#fcfcfb; --surface-2:#f7f5ef; --surface-3:#efece3;
  --ink:#15110b; --ink-2:#57514a; --muted:#8b8478; --hairline:#e3ddd0; --rule:#d8d1c1;
  --ring: rgba(21,17,11,.10);

  /* categorical identity — 8 fixed-order hues (validated, light surface) */
  --c1:#2a78d6; --c2:#1baf7a; --c3:#eda100; --c4:#008300;
  --c5:#4a3aa7; --c6:#e34948; --c7:#e87ba4; --c8:#eb6834;

  /* sequential magnitude — single blue ramp */
  --seq-1:#cde2fb; --seq-2:#86b6ef; --seq-3:#3987e5; --seq-4:#2a78d6; --seq-5:#1c5cab; --seq-6:#0d366b;

  /* diverging — blue↔red, gray midpoint (for Compare Δ) */
  --neg:#e34948; --neg-2:#f6c9c8; --mid:#e3ddd0; --pos-2:#bcd4f6; --pos:#2a78d6;

  /* status — fixed, icon+label, reserved meaning */
  --good:#0ca30c; --warn:#fab219; --serious:#ec835a; --crit:#d03b3b;

  --radius:12px; --radius-sm:8px;
  --font-display:'Fraunces', Georgia, serif;
  --font:'IBM Plex Sans', system-ui, sans-serif;
  --mono:'IBM Plex Mono', ui-monospace, monospace;
}
```

> Palette validated with the dataviz skill's `validate_palette.js`:
> **PASS** (exit 0) light + dark. Light has a contrast WARN on slots 2/3/7
> (aqua/yellow/magenta < 3:1) — mitigated by legend + direct labels + the
> ever-present data table (relief channel). Categorical hues are assigned in
> fixed slot order, never cycled; a 9th series folds into "Other" or small multiples.

### Typography

| Use | Face | Notes |
|---|---|---|
| Brand title, page H1, hero figure | `Fraunces` (display serif) | chrome only — **never in a chart** |
| UI body, all chart text | `IBM Plex Sans` | chart titles, axes, labels, tooltips |
| Scheme codes, figures, table numbers | `IBM Plex Mono` | `tabular-nums` for aligned columns |

### Chart conventions (Chart.js 4)

- **Form by the data's job** — magnitude→bar/heatmap; identity→grouped/stacked bar
  or multi-line; trend→line; part-to-whole→stacked bar (horizontal for many/long
  categories); polarity→diverging bar (Compare deltas).
- Thin marks, 2px lines, ≥8px markers, recessive grid/axes (hairline `--hairline`).
- **One axis per chart.** Two measures of different scale → two charts, not dual-axis.
- Legend for ≥2 series; ≤4 also direct-labeled; identity never color-alone.
- Text wears ink tokens (`--ink`,`--ink-2`,`--muted`), never the series color.
- Stacked segments + adjacent bars get a 2px surface gap (`--surface`).
- Hover: crosshair+tooltip on line/area; per-mark tooltip on bar/dot/cell.

### Motion (CSS-only)

Staggered `fade-up` on tab enter (cards/tiles via `animation-delay`), hover lift on
cards, smooth chart redraw. No JS animation libs.

## 6. Layout

```
┌──────────────────────────────────────────────────────────┐
│ Sidebar │ Topbar: title · [snapshot][scheme][month][mode] │
│ (tabs)  ├──────────────────────────────────────────────────┤
│         │ Tab content                                       │
│         │   · charts grid (top)                             │
│         │   · data table (below)                            │
└─────────┴──────────────────────────────────────────────────┘
```

Sidebar = tab list (Summary + 11 detail). Topbar = global filter bar (shared state).
Each detail tab renders into the same content slot: a chart grid then a table.

## 7. Drill-through (Summary → detail)

Summary charts are clickable. Clicking dispatches a nav event carrying
`{ tab, filter }` (e.g. trustee tab pre-set to one trustee). The target tab reads
the filter on mount and applies it. This is the only cross-tab coupling.

## 8. Accessibility

- Every chart has a `role="img"` + `aria-label` title; the data table is the
  non-color alternative (relief channel).
- Color contrast: ink tokens pass WCAG AA on the surface; status colors ship with
  icon+label.
- Keyboard: tab list is arrow-navigable; filter controls are native `<select>`/buttons.
- `prefers-reduced-motion` disables the stagger/hover motion.
